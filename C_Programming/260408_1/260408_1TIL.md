# 260408 C언어 트러블슈팅: 초과근무 수당 중복 계산 및 정수 연산

## 1. 문제 요약
기본 시급과 근무 시간을 입력받아 최종 급여를 계산하는 파이프라인 구축.

**[처리 조건]**
* 8시간까지는 기본 시급으로 계산
* 8시간 초과 근무는 기본 시급의 1.5배로 계산
* 총 급여가 100,000원 이상이면 보너스 5,000원 추가

---

## 2. 초기 코드 (The Trap)
```c
#define _CRT_SECURE_NO_WARNINGS
#include <stdio.h>

int main()
{
	int time, pay;
	int total_pay;

	scanf("%d %d", &pay, &time);

	total_pay = pay * time;

	if (time > 8) {
		total_pay = total_pay + ((pay * 15 / 10) * (time - 8));
	}

	if (total_pay >= 100000) {
		total_pay = total_pay + 5000;
	}

	printf("최종 급여 : %d", total_pay);

	return 0;
}
```

## 3. 문제점 분석 (Logical Flaw)
total_pay = pay * time; 구간에서 이미 전체 근무시간이 기본 시급으로 계산되어 누적되어 있다.

그런데 초과근무 시간에 대해 다시 1.5배 전체를 더하면, 초과 시간의 기본급이 중복 계산되는 치명적 논리 오류가 발생한다.

에지 케이스 검증 (Dry Run): 시급 10,000원, 근무시간 10시간

기본 계산: 10,000 * 10 = 100,000 (여기에는 초과 2시간의 기본급도 이미 포함됨)

여기에 초과 2시간의 1.5배를 또 더하면 초과분이 과하게 계산된다.

해결책: 이 파이프라인(가산 방식)에서는 초과 시간에 대해 추가 0.5배만 더해야 한다.

## 4. 최종 최적화 코드 (Final Verified Code)
```c
#define _CRT_SECURE_NO_WARNINGS
#include <stdio.h>

int main()
{
	int time, pay;
	int total_pay;

	scanf("%d %d", &pay, &time);

	// 1. Base Pipeline: 전체 시간에 대한 기본급 선지급
	total_pay = pay * time;

	// 2. Surge Pipeline: 8시간 초과분에 대한 0.5배 가산 (정수 연산 적용)
	if (time > 8) {
		total_pay = total_pay + ((pay * 5 / 10) * (time - 8));
	}

	// 3. Bonus Pipeline: 최종 금액 기준 보너스 지급
	if (total_pay >= 100000) {
		total_pay = total_pay + 5000;
	}

	printf("최종 급여 : %d", total_pay);

	return 0;
}
```

## 5. 핵심 인사이트 (Takeaway)
총합 변수(Accumulator)의 상태 추적: 변수에 이미 어떤 값이 들어 있는지 메모리 관점에서 먼저 확인해야 한다. 이미 포함된 값을 고려하지 않고 수식을 덮어쓰거나 더하면 중복 계산 오류가 발생한다.

아키텍처 설계 방향: 초과근무 계산 시 '전체 재계산 방식'으로 갈 것인지, '추가분만 더하는(가산) 방식'으로 갈 것인지 뼈대를 먼저 확실히 정해야 한다.

부동소수점 오차 원천 차단: 돈 계산에 1.5f 같은 실수를 쓰면 메모리 한계로 정밀도 오차가 발생한다. pay * 5 / 10처럼 철저한 정수 연산을 통해 소수점 발생 자체를 통제해야 한다.
