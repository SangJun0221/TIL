# 260410_2 C언어 트러블슈팅: count 기반 과목 판정 및 파이프라인 분리

## 1. 문제 요약
3과목 점수를 입력받아 기준 점수(80점) 이상인 과목 수를 계산하고, 그 개수에 따라 최종 등급을 판정하는 로직 구현.

**[처리 조건]**
* 80점 이상인 과목 수를 센다.
* 3과목 80점 이상: `최우수`
* 2과목 80점 이상: `우수`
* 1과목 80점 이상: `보통`
* 0과목 80점 이상: `노력 필요`

---

## 2. 초기 구현 및 최종 코드 (Verified Code)
이번 로직은 초기 설계부터 결함 없이 요구사항을 완벽하게 충족했다.
```c
#define _CRT_SECURE_NO_WARNINGS
#include <stdio.h>

int main()
{
	int score1, score2, score3;
	int count;

	scanf("%d %d %d", &score1, &score2, &score3);
	count = 0; // 카운터 초기화

	// [Phase 1: 독립적 조건 검사 및 카운팅]
	if (score1 >= 80) {
		count++;
	}
	if (score2 >= 80) {
		count++;
	}
	if (score3 >= 80) {
		count++;
	}

	// [Phase 2: 최종 상태(count) 기반 판정 및 출력]
	if (count == 3) {
		printf("최우수\n");
	}
	else if (count == 2) {
		printf("우수\n");
	}
	else if (count == 1) {
		printf("보통\n");
	}
	else {
		printf("노력 필요\n");
	}

	return 0;
}
```

## 3. 아키텍처 분석 (Why it works)
이번 코드는 별도의 에러나 수정 과정이 필요 없었다. 그 이유는 **'데이터 수집(Counting)'**과 **'결과 판정(Evaluation)'**이라는 두 가지 로직을 파이프라인 상에서 완벽하게 분리했기 때문이다.

독립적 if문의 활용: else if로 묶어버리지 않고 3개의 if문을 독립적으로 배치하여, 3과목 모두 개별적인 검사를 누락 없이 통과할 수 있도록 설계한 점이 핵심이다.

상태(State) 관리: 조건이 맞을 때마다 즉각적으로 텍스트를 출력하지 않고, count라는 변수에 상태를 저장해둔 뒤 마지막에 단 한 번만 판정했다. 이는 시스템의 확장성과 유지보수성을 높이는 정석적인 패턴이다.

## 4. 핵심 인사이트 (Takeaway)
Counting 패턴의 정석: 횟수나 개수를 기반으로 판정해야 하는 비즈니스 로직에서는, 각 조건을 반드시 독립적인 if문으로 분리하여 데이터가 정확히 누적되도록 해야 한다.

관심사의 분리 (Separation of Concerns): 조건 검사 단계(Phase 1)와 최종 출력 분기 단계(Phase 2)를 분리해서 생각하고 코딩하는 습관이 코드의 안정성을 결정한다.

## 5. Action Item (다음부터 체크할 것)
[ ] 로직 설계 시, '데이터를 가공/수집하는 구간'과 '결과를 출력하는 구간'이 혼재되어 있지 않은지 구조 확인하기.

[ ] 여러 조건을 검사할 때, 하나만 만족하면 끝나는 로직(else if)인지, 독립적으로 모두 검사해야 하는 로직(독립 if)인지 손계산(Dry Run)으로 먼저 판별하기.
