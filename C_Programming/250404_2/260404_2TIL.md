예제
[실전 코딩 문제: 도쿄 넷카페(PC방) 누진 요금 계산기]
요구사항: 일본의 한 넷카페는 이용 시간에 따라 시간당 요금이 저렴해지는 누진 요금제를 채택하고 있다. 사용자가 이용한 시간(정수)을 입력하면, 총 지불해야 할 요금(엔)을 계산하여 출력하는 프로그램을 작성하라.
요금표 (누진제 적용):
처음 2시간까지: 시간당 500엔
2시간 초과 ~ 5시간 이하: 초과한 시간에 대해 시간당 400엔
5시간 초과: 초과한 시간에 대해 시간당 300엔

초기에 짠 코드
// 초기 코드 중 일부
if (a <= 2) {
b = a * 500;
}
else if (a > 2 && a <= 5) {
b = 1000 + (a - 2) * 400;
}
else if (a > 5) {
b = 1000 + 4 * 300 + (a - 5) * 300;
}

설명
a는 시간 b는 요금이다. 2시간 이하일 경우 시간당 500엔이기에 입력값에 500을 곱한다.
2시간을 넘고 5시간이하일 경우 누진세가 시간당 400엔이 붙는다 그렇기에 일단 2시간이상 사용했다는 뜻이기에 2시간 최대 요금인 1000엔을 더한다 거기에 입력값에서 2시간을 빼고 추가 누진세 400엔을 곱한다.
5시간을 넘을 경우 방금과 동일하게 일단 1000엔 더하기 5시간 밑인 4를 더하고 300을 곱한다 그리고 입력값에서 5시간 미만이기에 5를 빼고 누진세 300엔을 더한다.

문제점
문제 1. 누진 구간 요금 계산 오류: 5시간 초과 구간(a > 5) 계산 시, 이전 구간(2~5시간)의 누적 요금을 4 * 300으로 잘못 산정함. (해당 구간의 최대 누적 시간은 3시간이며, 단가는 400엔이므로 3 * 400이 되어야 함)
문제 2. 조건식 연산 중복 (비효율성): else if (a > 2 && a <= 5)에서 이미 앞선 if (a <= 2)를 통과하지 못해 a > 2임이 확정되었음에도 불필요한 조건 검사를 수행함.
문제 3. 불필요한 조건문 사용: 마지막 else if (a > 5) 역시 앞선 조건을 모두 만족하지 않는 나머지 경우이므로 조건 명시가 불필요함.

수정한 코드
#define _CRT_SECURE_NO_WARNINGS
#include <stdio.h>

int main() {
int time, fee; // 직관적인 변수명으로 변경 (a -> time, b -> fee)

printf("이용 시간 입력: ");
scanf("%d", &time);

if (time <= 2) {
fee = time * 500;
}
else if (time <= 5) {
// a > 2 조건 생략: 상단 if문에서 걸러지지 않았으므로 무조건 2 초과임
fee = 1000 + (time - 2) * 400;
}
else {
// 5 초과 조건 생략: 상단 조건들을 통과하지 못한 나머지 모든 경우
// 누적 요금 수정: 기본 1000엔 + (3시간 * 400엔) + 초과분
fee = 1000 + (3 * 400) + (time - 5) * 300;
}

printf("총 요금: %d\n", fee);

return 0;
}

해결 및 깨달음
조건문의 폭포수(Waterfall) 흐름 이해: if - else if - else 구조는 위에서부터 순차적으로 실행되며 상호 배제(Mutual Exclusivity) 원칙이 적용된다. 상위 조건에서 걸러지지 않고 내려온 데이터는 이미 상위 조건의 여집합을 충족한다는 것을 의미하므로, 이를 활용해 불필요한 중복 조건(&&)을 제거하고 코드를 간결화해야 한다.
누진제 비즈니스 로직의 명확화: 각 요금 티어(Tier)별 누적 시간과 적용 단가를 명확히 분리하여 수식을 작성해야 계산 누락을 방지할 수 있다.
