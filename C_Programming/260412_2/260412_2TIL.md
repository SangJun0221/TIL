# 260412_2 C언어 트러블슈팅: 중첩 반복문(Nested Loop)의 공간 매핑과 인덱스 종속성

## 1. 문제 요약
입력받은 정수 `a`만큼의 높이를 가지는 직각 삼각형(Right Triangle) 형태의 별(`*`) 패턴을 출력하는 로직 구현.

**[처리 조건]**
* 첫 번째 줄에는 `*` 1개 출력
* 두 번째 줄에는 `*` 2개 출력
* `i`번째 줄에는 `*` `i`개 출력 (줄이 바뀔 때마다 별의 개수 증가)

---

## 2. 초기 구현 및 최종 코드 (Verified Code)
실행 결과는 요구사항을 충족했으나, 코드가 동작하는 근본적인 아키텍처(왜 맞는지)를 구조적으로 뜯어보고 0-Index 기반으로 최적화하는 과정을 거쳤다.

```c
#define _CRT_SECURE_NO_WARNINGS
#include <stdio.h>

int main()
{
	int a;
	int i, j;

	scanf("%d", &a);

	// [아키텍처 표준화: 0-Indexed Loop 적용]
	for (i = 0; i < a; i++) {           // Outer Loop: 행(Row, y축) 제어
		for (j = 0; j <= i; j++) {      // Inner Loop: 열(Column, x축) 제어
			printf("*");
		}
		printf("\n"); // 한 행의 출력이 끝나면 개행
	}

	return 0;
}
```

## 3. 아키텍처 분석 (Why it works)
이 코드가 정확히 직각 삼각형을 그리는 이유는 내부 루프의 종료 조건이 외부 루프의 현재 상태에 종속되어 있기 때문이다. 이를 '인덱스 종속성(Index Dependency)'이라 한다.

외부 반복문 (i): 2차원 공간의 Y축 (행, Row)

for (i = 0; i < a; i++)

콘솔창에서 줄바꿈을 담당한다. 루프가 한 번 돌 때마다 \n을 통해 다음 줄로 넘어간다.

내부 반복문 (j): 2차원 공간의 X축 (열, Column)

for (j = 0; j <= i; j++)

여기서 핵심은 종료 조건이 j < a가 아니라 **j <= i**라는 점이다.

i = 0 (1번째 줄)일 때: j는 0부터 0까지 ➔ 1번 실행

i = 1 (2번째 줄)일 때: j는 0부터 1까지 ➔ 2번 실행

i의 값이 커짐에 따라 내부 루프의 실행 횟수가 선형적으로 증가하며 직각 삼각형 모양이 렌더링된다.

## 4. 핵심 인사이트 (Takeaway)
공간 매핑 (Spatial Mapping): 콘솔 화면의 출력은 2차원 배열과 같다. 바깥쪽 루프는 'Y축(줄 수)', 안쪽 루프는 'X축(출력될 문자의 개수)'을 제어한다는 공간적 감각을 체화해야 한다.

함수적 사고 (f(x) = y): 별찍기 문제는 결국 i와 j의 관계식을 찾는 수학 문제다. 코딩하기 전에 종이에 i값의 변화에 따른 목표 별 개수를 적고, 이를 j의 조건식(j <= i 등)으로 도출하는 습관을 들여야 한다.

0-Index 습관화: C언어의 모든 배열과 메모리 오프셋은 0부터 시작한다. 1부터 세는 인간의 직관을 버리고, 반복문은 항상 for (i = 0; i < N; i++) 형태로 작성하여 미래의 메모리 접근 에러를 예방해야 한다.

## 5. Action Item (다음부터 체크할 것)
[ ] 중첩 반복문 문제를 만나면, 감으로 짜지 말고 i가 0, 1, 2일 때 필요한 문자의 개수를 표로 먼저 정리하기.

[ ] 내부 루프(j)의 한계값이 상수인지, 외부 루프(i)에 종속된 변수인지 파악하기.

[ ] C언어의 모든 루프는 0부터 시작하는 것을 기본 원칙으로 삼기.
