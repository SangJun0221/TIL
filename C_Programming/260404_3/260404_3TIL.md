1. 프로젝트 목표
트래픽 증감에 따라 서버 대수를 조절하는 로직을 구현하며, 발생할 수 있는 **런타임 에러(Runtime Error)**를 사전에 차단하는 예외 처리 능력을 기른다.
예제
[실전 코딩 문제: AWS 서버 오토 스케일링(Auto Scaling) 시뮬레이터]
요구사항: 트래픽 증감에 따라 AWS 서버 인스턴스 개수를 조절하는 간단한 시뮬레이터를 작성하라. 현재 서버 대수(a), 변경할 수치(b), 그리고 조절 방식 연산자(c)를 입력받아 최종 서버 대수를 출력한다.
입력: 정수 2개와 문자 1개를 차례대로 입력받는다. (예: 5 2 +)
연산자 조건 및 출력:
'+' (스케일 아웃): 현재 서버 대수에 입력된 수치를 더하여 출력한다.
'-' (스케일 인): 현재 서버 대수에서 입력된 수치를 뺀다.
[예외 처리 필수] 만약 뺀 결과가 1 미만(0이나 음수)이 된다면 연산을 수행하지 말고 "Error: 서버는 최소 1대 이상이어야 합니다."라고 출력하라.
'*' (트래픽 폭증): 현재 서버 대수에 입력된 수치를 곱하여 출력한다.
'/' (비율 축소): 현재 서버 대수를 입력된 수치로 나눈 몫만 출력한다.
[예외 처리 필수] 입력된 수치가 0일 경우 "Error: 0으로 나눌 수 없습니다."라고 출력하라.

초기에 짠 코드
#define _CRT_SECURE_NO_WARNINGS#include <stdio.h>int main()
{
char c;
int a, b;

printf("입력 (현재대수 수치 연산자): ");
scanf("%d %d %c", &a, &b, &c);

if (c == '+')
{
c = a + b;
}
else if (c == '-')
{
c = a - b;
}
else if (c == '/')
{
c = a / b;
if (a == 0 && b == 0)
{
printf("Error: 0으로 나눌 수 없습니다.");
}

}
else if (c == '*')
{
c = a * b;
}

if (c < 0)
{
printf("Error: 서버는 최소 1대 이상이어야 합니다.");
}

printf("%d", c);

return 0;
}

2. 초기 코드의 치명적 결함 (The Trap)
초기에는 단순히 연산자별 분기(if-else if)만 작성하였으나, 다음과 같은 심각한 버그들이 존재했다.
타입 오용 (Type Mismatch): 기호 저장용 char 변수에 계산 결과(int)를 담아 오버플로우 위험 발생.
런타임 에러 방치: 분모가 0일 때 에러 메시지만 띄우고 연산을 강행하여 프로그램이 강제 종료됨(Crash).
경계값 판정 오류: 서버가 1대 미만이 되는 상황을 정확히 차단하지 못함.

3. 단계별 디버깅 과정
STEP 1. 데이터 타입 및 예외 처리 위치 수정
결과값을 담을 전용 정수 변수 int result 추가.
연산 전 미리 검사(Action after Check)하는 구조로 변경.
STEP 2. 조기 종료(Early Exit) 도입
단순히 printf로 에러를 알리는 것을 넘어, **return 0;**을 사용하여 사고가 터지기 전 프로그램의 흐름을 즉시 끊어주는 '가드 절(Guard Clause)' 패턴 적용.

4. 최종 최적화 코드 (Final Verified Code)
#define _CRT_SECURE_NO_WARNINGS
#include <stdio.h>

int main() {
char op;
int current, value, result;

printf("입력 (현재대수 수치 연산자): ");
scanf("%d %d %c", &current, &value, &op);

// 1. 스케일 아웃 (덧셈)
if (op == '+') {
result = current + value;
}
// 2. 스케일 인 (뺄셈 + 예외 처리)
else if (op == '-') {
if (current - value < 1) {
printf("Error: 서버는 최소 1대 이상이어야 합니다.\n");
return 0; // 예외 발생 시 조기 종료
}
result = current - value;
}
// 3. 비율 축소 (나눗셈 + 예외 처리)
else if (op == '/') {
if (value == 0) {
printf("Error: 0으로 나눌 수 없습니다.\n");
return 0; // 런타임 에러 방지를 위한 강제 브레이크
}
result = current / value;
}
// 4. 트래픽 폭증 (곱셈)
else if (op == '*') {
result = current * value;
}
else {
printf("Error: 잘못된 연산자입니다.\n");
return 0;
}

// 모든 예외를 통과했을 때만 결과 출력
printf("최종 서버 대수: %d\n", result);

return 0;
}

5. 핵심 인사이트 (Takeaway)
Guard Clause (가드 절): 비정상적인 조건이 감지되면 return을 통해 즉시 함수를 빠져나와야 한다. 이는 코드의 깊이(Depth)를 줄이고 시스템의 안정성을 극대화한다.
Action after Check: 나눗셈의 분모 확인처럼, 치명적인 오류를 일으킬 수 있는 변수는 연산에 활용하기 직전에 검증해야 한다.
타입 안전성: 각 변수가 담을 데이터의 성격과 크기를 고려하여 char와 int를 엄격히 구분해 사용해야 한다
