1. 프로젝트 목표
[실전 코딩 문제: AWS S3 스토리지 과금 시스템]
요구사항: 사용자의 현재 스토리지 용량, 추가할 용량, 그리고 고객 등급을 입력받아 최종 청구 요금을 계산하라.

입력:현재 사용 용량(정수), 추가 용량(정수), 고객 등급(문자 1개, 'N'은 일반, 'V'는 VIP)을 공백으로 구분하여 차례대로 입력받는다. (예: 500 200 V)

처리 조건 (반드시 순서대로 파이프라인을 통과할 것)
[예외 처리 1] 추가 용량이 0 미만(음수)이면 "Error: 잘못된 용량입니다."를 출력하고 연산을 즉시 종료한다.
[예외 처리 2] 기존 용량과 추가 용량을 합친 '총 용량'이 1000GB를 초과하면 "Error: 할당 한도 초과입니다."를 출력하고 연산을 즉시 종료한다.
[기본 과금] 총 용량 1GB당 10원씩 계산하여 기본 요금을 산정한다.
[할인 공정] 고객 등급이 'V'인 경우, 요금에서 20%를 할인한다. (단, 소수점 이하는 버리고 정수로 처리하되 반드시 명시적 형변환을 사용할 것)
[최소 유지비 공정] (할인이 적용된 후의) 최종 요금이 5,000원 미만이면, 인프라 유지비 1,000원을 추가로 부과한다.

초기에 짠 코드
```c
#define _CRT_SECURE_NO_WARNINGS\

#include <stdio.h>

int main()
{
	int storage1, storage2;
	int total_storage;
	char a;

	printf("입력 : ");
	scanf("%d %d %c", &storage1, &storage2, &a);

	if (storage2 < 0) {
		printf("Error: 잘못된 용량입니다.");

		return 0;
	}
	else if (storage1 + storage2 > 1000) {
		printf("Error: 할당 한도 초과입니다.");

		return 0;
	}

	total_storage = (storage1 + storage2) * 10;

	if (a == 'V') {
		total_storage = total_storage * 0.8f;
	}

	if (total_storage < 5000) {
		total_storage = total_storage + 1000;
	}

	printf("최종 청구 요금 : %d", total_storage);

	return 0;
}
```

2. 초기 코드의 치명적 결함 (The Trap)
명시적 형변환(Explicit Casting) 누락 (Critical) : total_storage = total_storage * 0.8f;는 암시적 형변환이기에 명시적 형변환으로 보다 정확히 수정해야한다.
변수 네이밍의 퇴보 (Minor) : char a;경우 의미를 알 수 없는 알파벳이기에 등급이라는 grade로 변경해야 한다.

3. 단계별 디버깅 과정
total_storage = total_storage * 0.8f;를 (int)(total_storage * 0.8f); 수정.
char a;를 char grade;로 수정.

4. 최종 최적화 코드 (Final Verified Code)
```c
#define _CRT_SECURE_NO_WARNINGS
#include <stdio.h>

int main()
{
	int storage1, storage2;
	int total_storage;
	char grade; //변수 네이밍을 a가 아닌 등급 grade로 변경.

	printf("입력 : ");
	scanf("%d %d %c", &storage1, &storage2, &grade);

	if (storage2 < 0) {
		printf("Error: 잘못된 용량입니다.");

		return 0;
	}
	else if (storage1 + storage2 > 1000) {
		printf("Error: 할당 한도 초과입니다.");

		return 0;
	}

	total_storage = (storage1 + storage2) * 10;

	if (grade == 'V') {
		total_storage = (int)(total_storage * 0.8f); //total_storage = total_storage * 0.8f;은 암시적 형변환이기에 명시적 형변환으로 수정.
	}

	if (total_storage < 5000) {
		total_storage = total_storage + 1000;
	}

	printf("최종 청구 요금 : %d", total_storage);

	return 0;
}
```

5. 핵심 인사이트 (Takeaway)
암시적 형변환에 의존한 소수점 버림(total_storage * 0.8f)은 예상치 못한 데이터 손실을 야기할 수 있다. 향후 엄격한 정적 타입 시스템(Java 등)을 다루기 위해, 개발자의 의도를 컴파일러와 동료에게 명확히 전달하는 **명시적 형변환((int))**을 습관화해야 한다.
가독성은 곧 성능이다: 코드는 컴퓨터가 실행하지만, 유지보수는 사람이 한다. 변수명만 봐도 로직이 이해되도록 직관적인 네이밍을 생활화해야 한다.
